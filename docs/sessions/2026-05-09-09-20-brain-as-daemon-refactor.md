## What Was Done
- Architectural shift: daemon owns a `claude --resume`-style long-running subprocess (the "brain") that does language work. Used to be raw fetch to api.anthropic.com with `ANTHROPIC_API_KEY`; now subscription auth via the user's OAuth credentials at `~/.claude/.credentials.json`. The brain has MCP tools to read belfry state and take user-visible actions (reply / deliver / decline). The daemon serializes calls to brain.send and falls back gracefully when the brain is down.

## Decisions & Trade-offs
| Decision | Rationale |
|----------|-----------|
| Full rewrite to brain-only (no API/CLI dual-backend) | User explicitly chose this. Public-app users without a Claude.ai subscription can still run belfry — language routes return "language layer is down", deterministic routes work. API-mode tabled as a future feature request rather than carried as a parallel codepath. Net code reduction: -1700 lines from deletes vs +1700 lines added in brain modules — net zero, but conceptually simpler (one auth path, one model interface). |
| Brain spawns at daemon startup, supervised throughout | Daemon IS the brain; user never needs a terminal. crash → respawn with exp backoff (1s→2s→…30s cap). Brain-down detection via isAlive() check; fallback message "language layer is down — deterministic routes still work" so the user is never silently ignored. |
| `claude --print --input-format=stream-json` (one-shot persistent) over `claude --resume` (multi-session-aware) | --print + stream-json keeps the subprocess alive across multiple stdin messages until stdin closes; no on-disk session persistence (`--no-session-persistence`). Conversation context accumulates in-memory for the brain's lifetime. Keeps things simpler than managing fixed session UUIDs. |
| Brain shares one process across summarize + classify calls | Tradeoff: context bleed risk between unrelated summarize calls vs simpler implementation + cheaper warm-cache reuse. System prompt explicitly tells the brain "each user message is independent unless the daemon references a prior turn" — Haiku largely complies. If context bleed becomes a real problem, can spawn a second brain dedicated to summarization. |
| Brain's MCP tool calls go through new daemon HTTP endpoints (`/brain/*`) | The brain MCP plugin (stdio) needs to dispatch back to the daemon to read state and take actions. HTTP loopback registry + same Bearer-token auth (the existing `~/.local/state/belfry/registry.token`). Same trust model as the existing `/register` etc. Late-binding `setBrainHandlers` setter on Registry because watcher + nicknames + brain are constructed in different orders during startup. |
| `--allowedTools mcp__belfrybrain__*` + `--permission-mode default` | Brain default-denies all Claude Code tools (Bash, Edit, Gmail/Drive/Calendar plugins, etc.) and only allows the 8 belfrybrain MCP tools. System-prompt soft constraint is now backed by hard policy. `--permission-mode default` + `--allowedTools` whitelist is safer than `bypassPermissions` (which would let the brain reach beyond belfry's surface). |
| MCP server name `belfrybrain` (no hyphen) | Claude transforms non-alphanumeric chars in server names to underscores in the tool prefix. Hyphenated `belfry-brain` would have produced `mcp__belfry_brain__*` tool names that don't match literal allow-list strings. Flat-alphanumeric server name avoids the transform entirely. |
| Kill child on turn timeout | Stale `.result` line for a timed-out turn would otherwise resolve a subsequent caller's pendingTurn with the old turn's text — visible symptom would be "Telegram reply has the wrong content." Killing the child forces a clean respawn through the normal backoff path; the dead process can't emit any more lines. |
| `stopping` re-check inside restart timer callback (in addition to clearTimeout in stop()) | Macrotask-queued timer can fire after stop() runs but before clearTimeout takes effect. Without the re-check, a deferred restart that loses the race resurrects the supervisor and spawns a zombie subprocess. Re-check ensures the timer callback no-ops when stopping is true. |
| agent-handler forwards brain's text response as fallback when non-empty | If the brain returns text (Haiku tool-compliance miss) instead of calling reply_to_telegram, user gets nothing. Forwarding the text means user is never silently ignored. Risk: occasional double-reply when both tool + text fire. Acceptable. |

## Solutions & Fixes
- **Brain initially output "I need to respond but am currently unable to access the session status tools due to permissions."** Fix: `--allowedTools` enumeration of the 8 belfrybrain MCP tools + `--permission-mode default` so non-listed tools stay default-deny. The brain knew about the tools (system prompt + MCP plugin connected) but every call was being default-denied because no TTY to prompt on with `--print`.
- **MCP server name `belfry-brain` produced unmatched `--allowedTools` strings.** Fix: rename to `belfrybrain` (no hyphen). Claude converts non-alphanumeric chars to underscores in tool name prefixes; a hyphenated server name would have produced `mcp__belfry_brain__*` tool names that don't match the literal allow-list strings.
- **Stale `.result` after timeout poisoning subsequent turns.** Fix: kill child on timeout. Two reviewers (bug-hunter + perf) flagged this independently as confidence-90 each — one of the two critical findings.
- **Restart timer race past stop() spawning zombies.** Fix: `if (this.stopping) return` re-check inside the timer callback (in addition to `clearTimeout` in stop()).
- **stdin EPIPE waiting 60s for the timeout when the brain just died.** Fix: write callback rejects pendingTurn immediately with the underlying error.
- **Misleading comment on brain-handlers.getSession** ("re-read fresh from disk") when impl just reads the watcher's lastSeen cache. Fix: corrected comment to reflect reality (no disk fallback).
- **agent-handler swallowing brain's text return** when the brain didn't call a tool. Fix: forward as a fallback reply. Risk of occasional double-reply accepted as better than silence.
- **JSON.stringify wrapping the user text in CLASSIFY prompt** produced `User said: "..."` with literal quotes around the user's words. Fix: drop the wrapper.
- **`registry.setBrainHandlers` constructor-merge bug** where my edit accidentally split the constructor body — `Registry.stop()` then crashed with "Cannot read properties of undefined". Surfaced via `belfry-brain-mcp.test.js` failing in sequential test run. Fix: restored the missing block.

## Key Learnings
- **`claude --print --input-format=stream-json` works as the primary subprocess interface for subscription-auth language work.** Wire format is straightforward (`{type:"user", message:{role,content}}` in, sequence of `{type:"assistant"|"result"|...}` out, `result.result` is the final text). Tool use happens out-of-band via the brain's own MCP channel, not the supervisor's stdin/stdout. `--no-session-persistence` lets the brain stay alive across multiple stdin messages without pollution disk state.
- **`--bare` is incompatible with subscription auth.** The CLI help is explicit: "Anthropic auth is strictly ANTHROPIC_API_KEY or apiKeyHelper via --settings (OAuth and keychain are never read)" with `--bare`. So we can't use --bare to skip plugin loading + skill catalog; the brain inherits all the user's plugins/skills. Cost: ~44KB of dynamic system prompt sections per turn (~5¢ on first cold turn, cheap on cache reads). Mitigation: trust the system prompt + `--allowedTools` whitelist to keep the brain on-task.
- **Claude Code's permission gate default-denies tool calls in `--print` mode.** No TTY → no human → no approval → deny. The brain wanted to call `reply_to_telegram` but got blocked silently, falling back to text output. The fix is `--allowedTools` enumeration. Without it, the brain works as a chatbot but can't drive belfry's actions.
- **MCP server names: claude transforms non-alphanumeric chars to underscores in the tool prefix.** Server `belfry-brain` → tools `mcp__belfry_brain__*`; server `claude.ai Gmail` → tools `mcp__claude_ai_Gmail__*`. Server `plugin:telegram:telegram` → tools `mcp__plugin_telegram_telegram__*`. For an --allowedTools allow-list to match, server names must be flat-alphanumeric or the allow-list strings must be transformed to match.
- **Subscription auth path is `~/.claude/.credentials.json` with `claudeAiOauth.accessToken`** carrying scopes including `user:inference`. Credentials.json mode 0600. Token rotates / has `expiresAt`; Claude Code itself manages refresh.
- **Late-binding setter pattern for Registry options.** `Registry.brainHandlers` can't be passed at construction because dependencies are built later. Adding `setBrainHandlers(handlers)` post-construction is cleaner than forward-declaring everything. Same pattern would work for any other late-built dep.
- **Stale stream-json result is a real correctness risk for any long-running subprocess interface with cancellation.** The race: turn A times out at T=60s, caller bails, turn B starts at T=61s, brain finally emits A's result at T=62s — gets matched against B's pendingTurn. Killing the child on timeout is the simplest fix; alternative would be turn-IDs in messages.
- **node --test test/ hangs around the digest tests when 30+ files run together with subprocess-spawning brain tests.** Each file passes alone, any 2-3 combined pass. Documented as a node-test-runner artifact at scale (likely concurrency limit + subprocess-cleanup-timing); not a code-correctness issue. Workaround: run files in batches.
